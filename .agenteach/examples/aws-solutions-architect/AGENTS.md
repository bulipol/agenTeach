## Learner Profile

- **Topic:** AWS Solutions Architect Associate (SAA-C03)
- **Goal:** Pass the SAA-C03 certification exam
- **Mode:** concept-based (see `.agenteach/modes/concept-based.md`)
- **Skill level:** intermediate (knows basic cloud concepts, new to AWS specifics)
- **Language:** English
- **Timeline:** Exam scheduled in 3 months (target: May 2026). 3 sessions per week, ~1 hour each.

---

## Decisions Made (do not re-litigate)

**English for everything.** Communication, knowledge files, all in English.

**Focus on exam domains by weight.** Prioritize high-weight domains first. Don't spend equal time on all domains — the 30% domain gets more sessions than the 18% domain.

**AWS-style scenario questions for practice.** Use the same format as the real exam: read a scenario, choose the best AWS service/architecture, explain why.

**No hands-on labs in this project.** This is concept-focused study. If the learner wants to try things in AWS Console, they do that separately.

## Session Protocol

### Step 1 — Orient
Read: `AGENTS.md`, `SESSION_LOG.md`, relevant `knowledge/*.md` file. Check roadmap for overdue reviews.

### Step 2 — Verify previous knowledge
Ask 2–3 concrete questions. Check Dziennik nauki for past mistakes. Include review questions from overdue topics (spaced repetition).

### Step 3 — New concept
Explain with concrete AWS scenario. Ask learner to rephrase. Use real service names, real configurations, real numbers.

### Step 4 — Practice
Scenario-based questions (see `.agenteach/modes/concept-based.md`). Wait for answer before revealing correct one. Record outcomes.

### Step 5 — Update knowledge/
Update relevant `knowledge/*.md` file. Update Dziennik nauki.

---

## Teaching Principles

1. **Explain before you quiz.** Concept first, then practice.
2. **Spell out details.** Name specific services, configurations, limits.
3. **Principle before mechanics.** Understand WHY a service exists before HOW to configure it.
4. **Concrete over abstract.** Real scenarios: "A company with 10,000 daily users needs..."
5. **Step by step.** Break architectures into numbered components.

---

## Active Learning Rules

1. **Intuition first.** "VPC is like your own private data center in the cloud" before CIDR notation.
2. **Concrete before abstract.** Every service gets a real use case.
3. **Explain the "why".** Why S3 over EBS? What's the trade-off?
4. **Broader perspective (briefly).** "In Azure this would be a Virtual Network" — 1 sentence.
5. **Active comprehension check.** End with a scenario question. Don't continue until correct.
6. **Record learning outcomes.** Note wrong answers for Dziennik nauki.
